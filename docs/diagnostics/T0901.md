# T0901: result propagation error

A Result error reached a boundary where it had to become diagnostic.

Handle the error arm explicitly or let `!` propagate through a callable whose result shape can carry that error.

`!` is shorthand for normal `Result` flow. It can only propagate from a callable
whose return shape can carry the error:

```tn
let parse_score(raw: String): Result<Int, String> = {
  let value = parse.int(raw)!
  Ok(value)
}
```

At a runtime boundary, an unhandled `Error(...)` becomes a diagnostic. If that is
not what you want, match it explicitly:

```tn
let value = match parse.int(raw) {
  Ok(item) => item
  Error(message) => 0
}
```
