# T0301: shape mismatch

A value's compile-time meaning does not satisfy the expected shape.

Use the diagnostic facts to see where Tune learned the expected and actual shapes, then adjust the expression, annotation, or constraint.

This is about meaning, not just runtime data. For example, if a callable promises
`Int`, every continuing path must produce something that can become `Int`.

```tn
let score(okay: Bool): Int = if okay => 1 else "retry"
```

Fix it by making the branches agree:

```tn
let score(okay: Bool): Int = if okay => 1 else 0
```

Or by admitting the union if both meanings are intentional:

```tn
let score(okay: Bool): Int | String = if okay => 1 else "retry"
```
