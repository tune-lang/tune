# T0601: self state error

A receiver-state operation violates Tune's struct state rules.

Use member calls and receiver fields through the state path Tune can prove. Avoid relying on hidden aliasing or mutation outside the receiver model.

Ordinary struct fields own their values. They cannot be used to create a
persistent ownership cycle:

```tn
struct Node {
  next: Node?
}

let node: Node = Node { next = none }
node.next = node
```

Use an explicit handle, resource, or future weak reference for graph
back-references. This keeps Tune's default storage path compatible with
stack/direct-drop/RC-erased execution instead of requiring a hidden cycle
collector.
