# T0804: match hole fallback

`_` is a hole pattern, not a catch-all fallback; use `else`.

Use `_` only where a name or shape is intentionally absent. Use `else` for the fallback match arm.

Use this:

```tn
match value {
  Ok(item) => item
  else 0
}
```

Not this:

```tn
match value {
  Ok(item) => item
  _ => 0
}
```

The distinction matters because `_` means “absent/unused meaning,” not “any
remaining case.”
